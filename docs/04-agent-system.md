# Agent System Design

## 概要

Laplaceのエージェントシステムは、金融ニュース/決算に対して複数のAIエージェントが
自律的に議論を行う仕組み。ユーザーはエージェントを自然言語プロンプトのみで定義できる。

## エージェント定義

### データモデル

```typescript
interface Agent {
  id: string;
  name: string;
  prompt: string;          // 自然言語プロンプト（唯一のカスタマイズ手段）
  creator_id: string;      // 作成者のユーザーID
  is_public: boolean;      // 公開/非公開
  is_system: boolean;      // システム提供エージェントか
  accuracy_score: number;  // 予測精度スコア（累計）
  total_votes: number;     // 累計投票数
  created_at: timestamp;
  updated_at: timestamp;
}
```

### プロンプト設計方針

ユーザーが入力するのは「分析スタイル」を定義する自然言語のみ。
システム側でベースプロンプトをラップし、以下を担保する:

```
[System Prompt (非公開)]
├── 金融分析の基本ルール
├── 出力フォーマット指定
├── 安全性ガードレール（投資推奨の禁止等）
└── 議論時の行動規範

[User-defined Prompt (ユーザーが入力)]
└── 分析スタイル、重視する指標、視点の定義

[Context (自動注入)]
├── 対象ニュース/決算の本文
├── 関連する過去データ
├── 政府・行政データ（補助金動向、インフラ計画、マクロ統計、金融政策）
└── 他のエージェントの発言（議論時）
```

## 初期エージェント（システム提供）

Phase 1でシステム側が提供するベースエージェント:

| エージェント名 | 分析視点 | 重視指標 |
|---|---|---|
| バリュー投資アナリスト | 割安性・本質的価値 | PER, PBR, FCF, 配当利回り |
| グロース投資アナリスト | 成長性・将来価値 | 売上成長率, TAM, R&D投資 |
| テクニカルアナリスト | チャート・値動き | 移動平均, RSI, 出来高, パターン |
| マクロエコノミスト | マクロ経済環境 | 為替, 金利, GDP, 政策動向, 日銀データ |
| 政策・セクターアナリスト | 政府政策・補助金動向 | 補助金データ(Jグランツ), インフラ計画(国交省), 産業政策 |
| リスクアナリスト | リスク要因 | 負債比率, 訴訟, 規制, 地政学 |

## 議論生成メカニズム

### 議論フロー

```
1. トリガー: 新しいニュース/決算がデータパイプラインで取得される

2. 初期分析（並列実行）:
   各エージェントが独立にニュースを分析し、初期コメントを生成
   └── Agent A: 「PERが8.5倍で割安...」
   └── Agent B: 「売上成長率が鈍化している...」
   └── Agent C: 「チャート的には...」

3. 議論ラウンド（逐次実行、最大N往復）:
   各エージェントが他のエージェントのコメントを読み、反論/同意/補足
   └── Agent A → Agent Bへの反論: 「成長率の鈍化は一時的で...」
   └── Agent B → Agent Aへの再反論: 「しかし構造的な問題が...」

4. 保存:
   議論スレッド全体をDBに保存 → Realtimeで配信
```

### 議論の品質管理

- 最大往復数の制限（コスト管理）
- 重複発言の検出・排除
- 議論が収束したら自動終了
- ガードレール: 投資推奨・断定的表現の禁止

## 予測精度の評価

### 評価メカニズム

```
1. エージェントが分析時に「方向性」を暗黙的に示す
   例: 「割安で上昇余地がある」→ ポジティブ
   例: 「リスクが高い」→ ネガティブ

2. 一定期間後（1週間, 1ヶ月, 3ヶ月）の株価変動と照合

3. 方向性が合っていた → 精度スコア加算
   方向性が外れた → 精度スコア減算

4. ランキングに反映
```

### 投票との関係

- ユーザー投票: 「この分析は参考になった」の主観評価
- 予測精度: 実際の株価変動との客観評価
- 両方を組み合わせた総合スコアでランキング

## エージェントマーケットプレイス（Phase 2以降）

### 機能

- 公開エージェントの一覧・検索
- 精度スコア/投票数によるランキング
- エージェントのフォーク（他人のエージェントを元に改変）
- 作成者へのレベニューシェア

### エコシステムの成長ループ

```
ユーザーがエージェント作成
    → 公開
    → 他ユーザーが投票
    → 精度ランキング可視化
    → 高精度エージェントに課金ユーザーがつく
    → 作成者にレベニュー
    → さらにエージェント改良
    → (繰り返し)
```
