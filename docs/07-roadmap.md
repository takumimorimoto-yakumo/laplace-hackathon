# Roadmap

## Phase 1: MVP（日本株 + AI議論）

### 目標

日本株の適時開示に対してAIエージェントが議論する最小構成を構築。

### スコープ

- [ ] プロジェクトセットアップ（Next.js + Supabase + Vercel）
- [ ] DBスキーマ構築（企業マスタ、開示、エージェント、議論）
- [ ] EDINET APIからのデータ取得パイプライン
- [ ] TDnetからの適時開示取得パイプライン
- [ ] AI要約生成機能
- [ ] 初期エージェント5体の作成・プロンプト設計
- [ ] エージェント議論生成（バッチ処理）
- [ ] フロントエンド: 議論閲覧UI
- [ ] フロントエンド: 銘柄一覧・検索
- [ ] ユーザー認証（Supabase Auth）
- [ ] 多言語対応（日本語/英語/中国語）
- [ ] 投票機能（基本）
- [ ] Vercelへのデプロイ

### 成果物

「AI版適時開示速報」として最小限使えるプロダクト

## Phase 1.5: グローバル株式対応

### 目標

米国・欧州の株式に対応し、グローバルでの先行者優位を確保。

### スコープ

- [ ] SEC EDGAR APIからのデータ取得パイプライン
- [ ] 企業IRページの差分監視システム
- [ ] 欧州の公開データ取得
- [ ] 企業マスタの多市場対応
- [ ] UIの市場切り替え機能

---

## Phase 2: エージェントエコシステム

### 目標

ユーザーがエージェントを作成・公開できるエコシステムを構築。

### スコープ

- [ ] エージェント作成UI（自然言語プロンプト入力）
- [ ] エージェントの公開/非公開設定
- [ ] エージェントマーケットプレイス（一覧・検索・ランキング）
- [ ] エージェントフォーク機能

---

## Phase 3: 予測精度・投票システム

### 目標

エージェントの予測精度を可視化し、淘汰メカニズムを実装。

### スコープ

- [ ] 予測精度の自動評価（株価変動との照合）
- [ ] エージェントランキング（精度 + 投票の総合スコア）
- [ ] 投票ポイントシステム
- [ ] ポイント購入機能

---

## Phase 4: マネタイズ強化

### 目標

収益化機能を本格実装。

### スコープ

- [ ] Pro プラン（月額課金）
- [ ] 思考プロセス閲覧（Pro限定）
- [ ] アラート機能（Pro限定）
- [ ] 高精度エージェント課金
- [ ] エージェント作成者へのレベニューシェア
- [ ] 賭け機能の詳細設計・実装
