## 確定済みの方針

### データ取得戦略

| 方針 | 内容 |
|---|---|
| Tier 1（企業データ） | 企業IRページ（スクレイピング） + EDINET API v2（無料） |
| Tier 2（政府・行政データ） | Jグランツ MCP, 国交省 MCP, e-Stat MCP, BOJ MCP（全て無料） |
| Tier 3（市場データ） | 株価データ（Phase 1.5〜、取得元は要決定） |
| Tier 4（有料データ） | TDnet API（月7万円〜）への移行を選択肢として残す |
| スクレイピング方式 | LLMによるパーサーレス構造化（企業ごとにパーサー不要） |
| 初期対象 | 日経225構成銘柄（225社） |
| ストレージ | Supabase（DB + Storage）に統合。外部ストレージ不使用 |

## リスク

| リスク | 影響度 | 対策 |
|---|---|---|
| LLMコストの増大 | 高 | バッチ処理で最適化、軽量モデル使用、差分検知でAPI呼び出し削減 |
| スクレイピングのブロック | 中 | robots.txt遵守、アクセス間隔確保、ブロック時は対象から除外 |
| IRページの構造変更 | 低 | LLMパーサーなので構造変更に自動対応 |
| 競合の出現 | 中 | 早期のグローバル展開で先行者優位 |
| 金商法上の問題 | 高 | 法的レビュー必須、免責事項の徹底 |
| Supabase の料金体系変更 | 低 | セルフホスト可能なOSS設計 |
| スクレイピングの取りこぼし | 中 | PMF確認後にTDnet API有料プランへ移行で解決 |
